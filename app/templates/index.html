Google Maps APIキー設定

作業手順
1. Google Cloudをお持ちでない場合: Googleアカウント作成 → Google Cloud Console にサインイン
2. 初回利用時は利用規約に同意し、請求先アカウント（課金）を作成する
3. 「プロジェクトを選択」→「新しいプロジェクト」でプロジェクトを作成する
4. 「APIとサービス」→「ライブラリ」で Places API を有効化する
5. 「APIとサービス」→「認証情報」→「認証情報を作成」→「APIキー」を作成する
6. 作成したキーの「APIの制限」で Places API のみ許可する
7. 必要に応じて「アプリケーションの制限」を設定する（最初は制限を弱めて疎通確認推奨）
8. 左メニューの「APIキー設定」でキーを入力して保存する
9. 「設定済み: xxxxxx...xxxx」が表示されることを確認する
10. 「データ取得」で「カフェ 東京」「自動車整備 横浜」などを指定して取り込みを実行する
11. 失敗時は監査ログとGoogle Cloudのエラー（課金、API制限、リファラ/IP制限）を確認する
よくあるエラーと対処法
REQUEST_DENIED
原因: Places API が有効化されていない OR 支払い方法が登録されていない
対処: Google Cloud Console → APIとサービス → ライブラリで「Places API」を有効化し、お支払いで請求先アカウントを確認する
PERMISSION_DENIED
原因: APIキーに「HTTPリファラ制限」または「IP制限」が設定されている
対処: Google Cloud Console → 認証情報 → APIキーをクリック → 「アプリケーションの制限」を確認し、テスト段階は「制限なし」に変更する
クロスオリジン要求がブロック
原因: CORS設定の問題（バックエンド側）
対処: アプリ管理者に確認 → 環境変数 CORS_ALLOW_ORIGINS に本番ドメインが含まれているか
その他のエラー
1. ブラウザで F12 → Network タブから /api/import/google-places をクリック
2. Response でエラーメッセージを確認
3. 監査ログでDEBUG情報を確認（要望時に追加可能）
